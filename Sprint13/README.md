## A. Фотокопии
Евлампия очень любит постить фотографии в инстаграм. 
Но она боится, что данные с сервера могут пропасть. 
Поэтому она приняла решения хранить все изображения также в Трешландских 
датацентрах в двух экземплярах.

Всего есть N датацентров, в каждом из которых можно разместить fi фотографий, 
где i = 0..N-1.

Но хранить две копии фотографии в одном датацентре ненадежно. 
Вдруг с ним что-нибудь случится. Поэтому так делать нельзя. 
Нужно определить, какое максимальное число фотографий Евлампия сможет сохранить.

В первой строке записано n - количество датацентров. Оно не превосходит 10000.
В следующей строке через пробел записаны n чисел - вместимости датацентров 
в штуках фотографий. Каждое из чисел не превосходит 10000.

Нужно вывести число, равное максимальному количеству фотографий, 
для которых можно хранить копии в этих датацентрах.

Примечания
Для решения нужно использовать жадный алгоритм.
Для примера приведено несколько жадных стратегий. 
Нет гарантии, что каждая из них являются верной для любых входных данных. 
Можно выбрать и реализовать одну из них, удостоверившись, что она является 
корректной, или предложить свой вариант.

Стратегия 1:
Выбрать два датацентра с наименьшей вместимостью и помещать в них столько 
фотографий, сколько возможно.

## B. Поиск в сломанном массиве
Алла ошиблась со структурой данных. Она решила хранить массив в кольцевом 
буфере.
Проблема в том, что массив был отсортирован. И в нем можно было искать 
элемент за логарифмическое время. Алла скопировала данные из кольцевого 
буфера а обычный массив. Но он больше не является отсортированным. 
Тем не менее нужно обеспечить возможность находить в нем элемент за O(logn).

Во второй строке записано положительное число k - искомый элемент.
n и k не превосходят 1000.
Далее в строку через пробел записаны n положительных чисел, 
каждое из которых не превосходит 1000.

Выведите индекс искомого элемента в массиве, если он найден. 
Иначе выведите -1.
